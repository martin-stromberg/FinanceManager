# Offene Aufgaben

Erstellt am: 2026-07-19
Abbruchgrund: Nachtraegliche Rueckmeldung zur Postenauflistung pro Berichtszeile

Die folgenden Aufgaben muessen in einem erneuten Lauf bearbeitet werden.

## Nicht budgetierte Posten deutlicher von budgetierten Posten trennen

- [ ] In der Auflistung der Posten pro Berichtszeile muessen nicht budgetierte bzw. passende, aber nicht gewertete Posten deutlicher von budgetierten Posten getrennt werden.
- [ ] Nicht budgetierte Posten sollen eine `opacity` von `0.8` erhalten.
- [ ] Die Darstellung soll weiterhin gut lesbar bleiben und nicht zu einem schwer lesbaren Grau-auf-Grau-Eindruck fuehren.

## Laden der Postenauflistung wieder beschleunigen

- [ ] Das Laden der Auflistung der Posten pro Berichtszeile dauert weiterhin zu lange.
- [ ] Die Auflistung war vor den letzten Aenderungen nahezu sofort sichtbar; dieses Antwortverhalten soll wieder erreicht werden.
- [ ] Die Loesung darf die fachliche Unterscheidung zwischen budgetierten und nicht budgetierten bzw. nicht gewerteten passenden Posten nicht verlieren.
- [ ] Vorhandene lokale Aenderungen am Rawdaten-Cache und an der Overlay-Darstellung sollen erhalten oder fachlich sauber ersetzt werden, soweit sie zur Anforderung passen.
